# 创业通产品战略文档

## 1. 产品定位
创业通是一款面向中国初创企业与创业者的全栈式智能服务平台，整合政策服务、融资对接、项目管理、资源匹配与知识赋能五大核心模块，打造"一站式创业操作系统"。

## 2. 商业价值分析

### 2.1 市场机会
- **政策红利未被有效触达**：地方政府每年投入超万亿创业扶持资金，但90%初创企业因信息不对称无法申请。
- **融资效率低下**：天使轮/Pre-A轮项目平均耗时4.2个月寻找合适投资机构，匹配成功率低于15%。
- **管理工具碎片化**：创业者需使用7+款工具完成项目管理、财务记录、团队协作，效率损失超30%。
- **知识获取成本高**：优质创业课程、法律模板、财税指南分散于公众号、知乎、付费平台，缺乏系统化整合。

### 2.2 目标用户
| 用户类型 | 需求痛点 | 付费意愿 |
|----------|----------|----------|
| 早期创业者（0-5人团队） | 不懂政策、不会写BP、找不到资源 | 高（节省时间=生存关键） |
| 创业孵化器 | 缺乏数字化管理工具、项目评估效率低 | 中高（提升入驻率与成功率） |
| 地方政府/园区 | 政策执行透明度低、补贴浪费严重 | 高（政绩+资金使用效率） |
| 天使投资人 | 项目筛选成本高、尽调信息不全 | 高（提升投资决策质量） |

### 2.3 收益模型
- **B2B2C订阅制**：向孵化器/园区收取SaaS年费（¥5,000–20,000/年）
- **增值服务**：BP优化、法律咨询、融资代运营（按项目收费）
- **数据服务**：匿名化创业数据报告出售给地方政府与研究机构
- **广告合作**：与律所、会计事务所、云服务商联合推广

## 4. 技术架构概览（服务端）
- **语言**：Go 1.24（高性能、高并发）
- **框架**：Gin（Web）、GORM（ORM）
- **数据库**：PostgreSQL（主库） + Redis（缓存）
- **搜索**：Elasticsearch（政策全文检索）
- **向量库**：Milvus（融资BP语义匹配）
- **AI服务**：本地部署Qwen3-30B-A3B（私有化部署保障数据安全）
- **异步任务**：Redis Queue + Worker Pool
- **认证授权**：JWT（JSON Web Token）用于用户身份验证
- **部署**：Docker + Kubernetes（阿里云ACK）

## 7. 系统架构详解

### 7.1 微服务架构
系统采用微服务架构设计，主要包含以下服务：
- **网关服务**: Gin框架提供统一入口，处理路由、鉴权、限流
- **政策服务**: 负责政策数据的获取、存储、检索
- **融资服务**: 处理BP上传、向量匹配、投资人推荐
- **帖子服务**: 负责帖子聚合、分类、推荐
- **用户服务**: 管理用户信息、权限控制
- **通知服务**: 处理系统通知、邮件、短信等

### 7.2 数据存储架构
- **关系型数据库**: PostgreSQL存储用户信息、政策详情、帖子等结构化数据
- **向量数据库**: Milvus存储BP、投资偏好等向量数据，支持语义相似度搜索
- **搜索引擎**: Elasticsearch提供全文检索能力，用于政策快速检索
- **缓存层**: Redis缓存热点数据，提升系统响应速度
- **文件存储**: 支持商业计划书等文件的分布式存储

### 7.3 AI与算法架构
- **文本向量化**: 使用预训练模型将BP、政策文本转换为向量
- **语义匹配**: 基于向量相似度实现BP与投资偏好的智能匹配
- **NLP处理**: 政策文本解析、关键词提取、内容摘要生成
- **推荐算法**: 基于协同过滤和内容推荐的混合推荐算法
